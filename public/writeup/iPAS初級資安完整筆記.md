## 第二章 | 科目一：資訊安全管理概論 核心考點

### 1. 資訊安全三要素 (CIA Model) — **必考基礎**

#### 定義與應用

| CIA 要素 | 定義 | 控制措施 | 常見破壞攻擊 |
|---------|------|--------|-----------|
| **機密性<br>(Confidentiality)** | 確保資料只有獲得授權的人才能存取 | 加密、存取控制、隔離 | 監聽(Sniffing)、駭客入侵、竊聽 |
| **完整性<br>(Integrity)** | 確保資料不被未經授權的更改，保持原始狀態 | 雜湊函數、數位簽章、資料庫備份 | SQL注入、XSS、資料篡改 |
| **可用性<br>(Availability)** | 確保使用者在需要時能及時存取與使用資源 | 備援、災難復原、防DDoS、負載平衡 | DDoS攻擊、系統當機、網路中斷 |

#### 試題判斷技巧

**常見題型**：「某攻擊破壞資安的哪項特性？」

- DDoS 大量封包 → **可用性** ✓
- 監聽竊取資料 → **機密性** ✓
- 資料庫遭SQL注入 → **完整性** ✓
- 官網首頁被置換 → **完整性** ✓

### 3. 相關法規概論與遵循 — **每年必考題**

#### 台灣重要法規

##### (1) 資通安全管理法

**事件分級與處理時間表**（必背 100%）：

| 事件等級 | 處理對象 | 通報時限 | 審核時限 | 處理時限 | 結案時限 |
|---------|--------|--------|--------|--------|--------|

**等級判定標準**：
- 1級：非核心資訊輕微洩漏
- 2級：非核心資訊嚴重洩漏
- 3級：核心資訊輕微洩漏或涉及關鍵基礎設施
- 4級：核心資訊嚴重洩漏、國家機密或關鍵基礎設施

**蒐集三要素**（告知義務）：
- 收集單位名稱
- 蒐集目的
- 蒐集資料類別

**賠償標準**（常考數字）：
- **每人每一事件**：500 元 ~ 20,000 元
- **單一事件上限**：2 億元

**隱私損害賠償原則**：
- 被害人無法證明實際損害時，法院得依傷害情節認定賠償金額

### 4. 資產與風險管理

#### 資產分類與盤點

**資產類型**：
- 資訊資產（資料庫、機密文件）
- 軟體資產（應用程式、系統）
- 硬體資產（伺服器、工作站）
- 人員資產（員工、承包商）

**資料擁有者 (Data Owner) 職責**：
- 決定資料分級
- 授權存取權限
- 負責資料安全

#### 風險評鑑與風險處理

##### 定量方法（數學計算）

**公式速記**：
```
SLE = 資產價值 (AV) × 暴露因子 (EF)
ALE = SLE × 年度發生率 (ARO)
```

**完整計算範例**：
- 資產價值 = 1,000,000 元
- 暴露因子 = 40%（發生損失時的損失比例）
- 年度發生率 = 0.02（每年發生機率 2%）
- SLE = 1,000,000 × 0.4 = 400,000 元
- ALE = 400,000 × 0.02 = **8,000 元**

##### 定性方法（等級評估）

使用「低、中、高」三級評估：
- 風險等級 = 發生可能性 × 衝擊程度
- 結果分類：低風險、中等風險、高風險

##### 風險處理四大策略

| 策略 | 定義 | 例子 |
|------|------|------|
| **降低 (Mitigate)** | 採取控制措施降低風險 | 安裝防火牆、進行備份 |
| **避免 (Avoid)** | 停止相關高風險活動 | 停止某項服務 |
| **轉移 (Transfer)** | 將風險轉給第三方 | 購買保險、外包給廠商 |
| **接受 (Accept)** | 承認並接受風險 | 記錄決策，監控風險 |

**提醒**：風險處理方案必須**真實可行**，不能只是「打嘴砲」！

### 5. 存取控制、加解密與金鑰管理

#### 身分認證 (Authentication) — **必背考點**

##### 認證三要素

| 要素 | 英文 | 例子 |
|------|------|------|
| **你知道的** | Something You Know | 密碼、PIN、祕密問題 |
| **你擁有的** | Something You Have | IC卡、USB token、行動電話 |
| **你是誰** | Something You Are | 指紋、虹膜、臉部辨識、掌形 |

**常見錯誤**：
- ✗ 「密碼」屬於「你是誰」 → 錯誤
- ✗ 「個人密碼」屬於多因素認證 → 錯誤（它是「你知道的」，不夠多因素）
- ✓ 「IC卡 + 隨機驗證碼」 → 正確（「你擁有的」+ 「你知道的」）

##### 認證 vs 授權

| 名詞 | 英文 | 含義 | 動詞形態 |
|------|------|------|---------|
| **身分驗證** | Authentication | 驗證你確實是本人 | 證明身份 |
| **授權** | Authorization | 給你相應的存取權限 | 給予權限 |

**記憶技巧**：
- Auth**N**entication = **N**ame (你是誰)
- Auth**O**rization = **O**wn (你擁有什麼權限)

#### 存取控制核心原則

| 原則 | 含義 | 例子 |
|------|------|------|
| **最小權限<br>(Least Privilege)** | 給予每人完成工作所需的最低權限 | 財務人員只能查看財務報表，不能查看人資檔案 |
| **僅知原則<br>(Need to Know)** | 限制對敏感資訊的存取權 | 機密檔案只有授權人員能看 |
| **職務區隔<br>(SoD, Separation of Duties)** | 防止單人同時擁有關鍵且互相衝突的權限 | 帳號申請人 ≠ 帳號核准人 ≠ 帳號建立人 |

---

#### 加密技術

##### 加密方式分類

| 加密類型 | 金鑰數量 | 用途 | 例子 | 速度 |
|---------|--------|------|------|------|
| **對稱式** | 1把(同一把) | 大量資料加解密 | AES, DES, 3DES | 快 ✓ |
| **非對稱式** | 2把(公私鑰) | 金鑰交換、簽章 | RSA, ECC | 慢 ✗ |

**非對稱加密特性**：
- 公開金鑰 (Public Key) → **可以公開讓任何人取得**
- 私密金鑰 (Private Key) → **絕對不能交給他人**

##### 雜湊函數 (Hash)

**用途**：檢驗資料完整性（產生指紋）

**常見演算法**：SHA2, SHA-256, MD5

**特性**：
- 輸入相同 → 輸出相同的指紋
- 輸入不同 → 輸出不同的指紋
- 無法反向解密（單向函數）

**試題常考**：
Q: 「檔案完整性檢查最常使用哪種技術？」
- ✗ ECC（用於加密）
- ✗ RSA（用於加密/簽章）
- ✓ **SHA2（雜湊函數）**
- ✗ AES（用於加密）

##### 數位簽章

**作用**：
1. 確認資訊來源（認證）
2. 確保資訊未被篡改（完整性）
3. 防止簽署人否認（不可否認性）

**流程**：
1. 對訊息進行雜湊
2. 用私密金鑰加密雜湊值 → 數位簽章
3. 收方用公開金鑰驗證簽章

---

#### 金鑰管理生命週期

**五個階段**：
1. **產生 (Generation)**：安全生成金鑰
2. **存放 (Storage)**：安全存儲（金鑰管理系統 KMS）
3. **使用 (Usage)**：正確使用金鑰
4. **撤銷 (Revocation)**：損壞或洩漏時立即撤銷
5. **銷毀 (Destruction)**：安全銷毀廢除的金鑰

### 6. 事故管理與營運持續

#### 事件 vs 事故 — **常考定義題**

| 名詞 | 定義 | 特徵 |
|------|------|------|
| **資訊安全事件<br>(Event)** | 系統/網路中任何偏離正常運作狀況 | 已識別、未必造成損害 |
| **資訊安全事故<br>(Incident)** | 事件已確實造成損害或威脅 | 實際造成負面影響 |

**區分標準**：依照對組織的**影響程度**和**嚴重性**

#### 日誌 (Log) 管理

**原則**：
- 記錄所有使用者活動、系統異常
- 依法定期限保存（通常 1 年以上）
- 定期檢查與備份
- **平日應持續監控**（不是事件發生時才查）

**常見題目**：
Q: 系統日誌為避免外洩，應在發生資安事件時才查詢，平日不應存取？
- 答案：**否** ✗（應該平日就持續監控）

---

#### 備援與災難復原

**關鍵指標**：
- **RTO (Recovery Time Objective)**：恢復時間目標（多久內恢復服務）
- **RPO (Recovery Point Objective)**：恢復點目標（可容忍的最大資料遺失）

**備份策略**：
- 完整備份：備份所有資料
- 增量備份：備份自上次備份後的變化
- 異地備份：在地理位置不同的地方備份（防止天災）

### 1. 網路與通訊安全

#### OSI 模型與常見協定

| OSI 層級 | 層名稱 | 常見協定/技術 | 防護工具 |
|---------|--------|-----------|---------|
| **7** | **應用層** | HTTP, HTTPS, DNS, SMTP, SSH | WAF (Web應用防火牆) |
| 6 | 表達層 | 加密、壓縮、編碼 | - |
| 5 | 會議層 | 連線建立/管理 | - |
| **4** | **傳輸層** | TCP, UDP | - |
| **3** | **網路層** | **IP, IPsec** | 防火牆 |
| 2 | 資料鏈結層 | Ethernet, PPP | - |
| 1 | 物理層 | 光纖、銅線、無線電波 | - |

**高頻考點**：
- IPsec → **第 3 層（網路層）** ✓

#### 網路防護設備

| 設備 | 功能 | 防護層級 |
|------|------|--------|
| **防火牆 (Firewall)** | 根據規則允許/阻止流量 | 網路層/應用層 |
| **IDS (入侵偵測系統)** | 監控異常流量、發送警報 | 被動監控 |
| **IPS (入侵防禦系統)** | 監控並主動阻止攻擊 | 主動防禦 |
| **WAF (Web應用防火牆)** | 保護Web應用，防止注入/XSS | 應用層 |

**常見誤解**：
- ✗ 防火牆能有效阻擋 DDoS → 錯誤（DDoS 是大量合法流量）
- ✓ IPS 能主動阻止攻擊 → 正確

### 3. 作業系統與應用程式安全

#### Web 應用常見漏洞 (OWASP Top 10)

##### (1) SQL 注入 (SQL Injection)

**原理**：輸入資料變成 SQL 指令，未做好輸入驗證

**例子**：
```sql
正常查詢：SELECT * FROM users WHERE id=123
攻擊：SELECT * FROM users WHERE id=123 OR 1=1 --
結果：返回所有使用者資料！
```

**防禦方式**：
- ✓ **Prepared Statement**（參數化查詢）
- ✓ **Stored Procedure**（預存程序）
- ✓ 輸入驗證與過濾
- ✗ 只加密資料庫（無法防止注入指令層的問題）

---

##### (2) 跨站指令碼攻擊 (XSS, Cross-Site Scripting)

**原理**：未過濾的使用者輸入被執行成 JavaScript

**三種類型**：
1. **儲存型 XSS (Stored)**：惡意代碼存入資料庫，所有使用者都受影響
2. **反射型 XSS (Reflected)**：惡意代碼在 URL 參數中，需要誘導使用者點擊
3. **DOM 型 XSS**：前端 JavaScript 直接修改 DOM

**例子**：
```html
輸入：<script>alert('hacked')</script>
結果：每個訪問該頁面的人都會看到彈窗
```

**防禦方式**：
- ✓ 輸出編碼（將 `<` 轉為 `&lt;`）
- ✓ 過濾 HTML 標籤
- ✓ 使用 `.innerText()` 而不是 `.innerHTML()`

---

##### (3) 跨網站偽造請求 (CSRF, Cross-Site Request Forgery)

**原理**：誘騙已登入使用者執行非預期的請求

**例子**：
```
使用者登入了銀行網站
訪問惡意網站，該網站自動向銀行發送轉帳請求
```

**防禦方式**：
- 驗證碼 (CAPTCHA)
- CSRF Token
- SameSite Cookie

---

##### (4) 安全設定疏失 (Security Misconfiguration)

**例子**：
- 預設密碼未改
- 不必要的服務開啟
- 錯誤的權限設定

---

##### (5) 身分驗證功能缺失 (Broken Authentication)

**例子**：
- 弱密碼政策
- 無會話超時
- 密碼未加密存儲

### 4. 資安維運技術

#### 惡意程式 (Malware)

| 類型 | 定義 | 傳播方式 | 特徵 |
|------|------|--------|------|
| **病毒 (Virus)** | 附著在程式上執行的惡意程式 | 需要宿主程式執行 | 依附宿主 |
| **蠕蟲 (Worm)** | 自動複製和傳播的程式 | 自我複製、無需宿主 | 快速傳播 |
| **木馬 (Trojan)** | 偽裝成合法程式的惡意程式 | 偽裝下載、執行 | 偽裝性強 |
| **勒索軟體<br>(Ransomware)** | 加密使用者檔案並勒索贖金 | 釣魚、漏洞 | 檔案被加密、要求付款 |
| **間諜軟體<br>(Spyware)** | 竊取使用者資訊的程式 | 捆綁軟體、廣告 | 監控使用者行為 |

---

#### 弱點掃描與管理

**流程**：
1. **識別漏洞**：使用掃描工具（如 Nessus）
2. **評分與優先化**：根據嚴重程度排序
3. **修補 (Patch)**：立即應用供應商提供的補丁
4. **驗證**：確認修補成功

**常見工具**：
- Nessus、OpenVAS、Qualys

---

#### 資料加密

**按時點分類**：

| 加密場景 | 說明 | 技術 |
|---------|------|------|
| **傳輸中加密<br>(In Transit)** | 資料在網路上傳輸時加密 | TLS/SSL, IPsec, VPN |
| **儲存中加密<br>(At Rest)** | 資料在磁碟上存儲時加密 | AES, TDE, 欄位加密 |

**常見技術**：
- **TDE (Transparent Data Encryption)**：資料庫層面透明加密
- **欄位加密**：針對敏感欄位（如信用卡號）加密
- **檔案加密**：整個檔案加密

---

#### 備份管理

**核心原則**：
- 定期備份（每日/每週）
- 異地儲存（防止天災或資料中心故障）
- 定期測試復原程序（確認備份可用）
- 記錄備份時間和內容

---

#### 日誌管理

**Syslog 協定**：
- 用於集中管理網路設備日誌
- 包含時間戳、設備、事件信息

**日誌分析方法**：
- 日誌聚合與關聯分析
- 與 IOC (Indicators of Compromise) 比對
- 識別異常行為

---

### 5. 新興科技安全

#### 雲端安全 (Cloud Security)

**服務模式**：

| 服務類型 | 說明 | 責任 | 典型例子 |
|---------|------|------|--------|
| **IaaS** | 基礎設施即服務 | 使用者負責：OS、應用、資料 | AWS EC2 |
| **PaaS** | 平台即服務 | 使用者負責：應用、資料 | Google App Engine |
| **SaaS** | 軟體即服務 | 使用者僅使用應用 | Microsoft 365, Salesforce |

**安全考量**：
- 不掌控作業系統、硬體或運作網路（在 SaaS 情況下）
- 依賴供應商安全措施
- 資料主權與合規性問題

---

#### 行動裝置安全

**身份識別碼**：
- **IMSI (International Mobile Subscriber Identity)**：行動用戶識別碼
- **IMEI (International Mobile Equipment Identity)**：行動設備識別碼
- **UUID (Universally Unique Identifier)**：通用唯一識別碼

**安全措施**：
- 行動應用程式安全檢測
- 行動設備管理 (MDM)
- 遠端鎖定/清除功能

---

#### 物聯網 (IoT) 與智慧製造安全

**特色風險**：
- 設備預設密碼未改
- 運算能力有限，難以執行複雜安全措施
- 大量互聯設備
- 持續連網易被利用

## 第五章 | 具體試題與解析

### 高頻經典試題

#### 【管理類】法規時間題

**113 年初級試題**

Q1: 主管機關於接獲資安事件通報後，若判定為 3 級或 4 級事件，應於幾小時內完成審核？

選項：
- (A) 一小時內
- (B) **2 小時內** ✓
- (C) 8 小時內
- (D) 24 小時內

**解析**：直接從法規時間表查詢，3-4 級審核時限 = 2 小時

---

#### 【管理類】ISO 27001:2022 改版

**113 年初級試題**

Q2: ISO 27001:2022 相較 2013 版本，增加了哪些安全管理用詞？

選項：
- (A) **Cybersecurity；Information Security** ✓
- (B) Privacy；Data Protection
- (C) Risk Assessment；Threat Analysis
- (D) Incident Management；Compliance

**解析**：2022 版新增網路安全 (Cybersecurity) 與資訊安全 (Information Security) 等用詞，強調應對現代威脅

---

#### 【管理類】風險計算

**常見試題**

Q3: 資訊資產價值 2,000,000 元，年度發生威脅機率 5%（0.05），威脅發生時損失率 60%。求年度損失預期值 (ALE)？

**解法**：
```
資產價值 (AV) = 2,000,000 元
暴露因子 (EF) = 60%
單一損失預期 (SLE) = 2,000,000 × 0.6 = 1,200,000 元
年度發生率 (ARO) = 0.05 (5%)
年度損失預期 (ALE) = 1,200,000 × 0.05 = 60,000 元
```

**答案**：60,000 元

---

#### 【技術類】CIA 要素判斷

**常見試題**

Q4: 公司官網首頁被駭客置換，此為資安三要素中哪項遭受破壞？

選項：
- (A) 機密性
- (B) **完整性** ✓
- (C) 可用性
- (D) 真確性

**解析**：官網內容被篡改（非授權更改）→ 完整性被破壞

---

#### 【技術類】SQL 注入防禦

**常見試題**

Q5: 下列何者最「不」能防範 SQL 注入攻擊？

選項：
- (A) **對資料庫進行加密** ✗
- (B) 對查詢字串進行字串過濾 ✓
- (C) 使用 Prepared Statement ✓
- (D) 使用 Stored Procedure ✓

**解析**：
- SQL 注入是「指令層」的攻擊
- 加密資料庫是「資料層」的保護，無法防止指令被竄改
- 真正的防禦是「輸入驗證」和「參數化查詢」(Prepared Statement)

---

#### 【技術類】應用程式檢測方法

**常見試題**

Q6: 下列何者「不」是評估應用程式安全性的檢測方法？

選項：
- (A) Penetration Testing（滲透測試）✓
- (B) **Ransomware Testing（勒索軟體測試）** ✗
- (C) Threat Modeling（威脅建模）✓
- (D) Source Code Review（原始碼審查）✓

**解析**：
- 應用程式安全檢測 = 尋找程式碼漏洞
- 勒索軟體測試 ≠ 應用程式安全檢測（它是惡意程式測試）

## 第六章 | 考試答題技巧與時間管理

### 閱讀與作答技巧

#### Step 1：快速掃描題目

閱讀題目時圈出關鍵詞：
- **「不」、「最」、「何者錯誤」** → 反向題
- **「正確」、「適切」** → 正向題

#### Step 2：題目分類

分類題目難度：
1. **簡單題**（5-10 分鐘內可做）：先做，建立信心
2. **中等題**（需思考 1-2 分鐘）：其次做
3. **難題**（需深度思考或計算）：最後做

#### Step 3：排除法

對於不確定的題目：
- 先排除明顯錯誤的選項
- 在剩餘選項中做出最佳判斷
- 不要空白不答

### 時間分配建議

**總時間**：75 分鐘，50 題

| 階段 | 時間 | 任務 |
|------|------|------|
| 第一遍 | 15 分鐘 | 做有把握的題目（30-35 題） |
| 第二遍 | 30 分鐘 | 做中等難度題目（10-15 題） |
| 第三遍 | 20 分鐘 | 處理難題或有疑問的題目 |
| 最後檢查 | 10 分鐘 | 確認是否有漏答、檢查計算 |

---

### 常見陷阱與避免方法

#### 陷阱 1：法規時間混淆

**常見錯誤**：
- ✗ 混淆「通報」、「審核」、「處理」、「結案」時間

**避免方法**：
- ✓ 背誦完整的時間表並多次重複
- ✓ 做題時立即查表，建立肌肉記憶

#### 陷阱 2：CIA 要素混淆

**常見錯誤**：
- ✗ DDoS 破壞完整性（誤選）
- ✗ 監聽破壞可用性（誤選）

**避免方法**：
- ✓ 理解而非死記：DDoS = 大量請求 = 系統無法服務 = **可用性**
- ✓ 監聽 = 竊取資訊 = **機密性**
- ✓ 資料篡改 = **完整性**

#### 陷阱 3：認證 vs 授權混淆

**常見錯誤**：
- ✗ 「密碼」屬於「授權」
- ✗ 「指紋」屬於「身分驗證」的「你知道的」

**避免方法**：
- ✓ Authentication = **身份驗證**（你是誰）
- ✓ Authorization = **權限授予**（你有什麼權限）
- ✓ 認證三要素 = 你知道的、你擁有的、你是誰（生物特徵）

#### 陷阱 4：加密技術混淆

**常見錯誤**：
- ✗ 用 RSA 驗證檔案完整性
- ✗ 用雜湊函數進行加密

**避免方法**：

| 技術 | 用途 | 不能用於 |
|------|------|--------|
| 對稱加密 (AES) | 大資料加解密 | ✗ 金鑰交換 |
| 非對稱加密 (RSA) | 金鑰交換、簽章 | ✗ 大資料加密 |
| 雜湊函數 (SHA2) | **完整性驗證** | ✗ 加密 |
| 數位簽章 | 認證、完整性、不可否認 | ✗ 大資料保護 |
